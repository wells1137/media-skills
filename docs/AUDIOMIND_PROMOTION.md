# Audiomind Skill Promotion Guide

How to help more OpenClaw users discover and use audiomind.

---

## 1. Optimize the ClawHub Listing (First Impression)

- **ClawHub link:** https://clawhub.ai/skills/audiomind  
- **Tips:**
  - When releasing a new version, write a clear `--changelog` (e.g. "terse replies / new model support") so visitors can decide whether to install.
  - Keep `--tags` broad: `audio`, `tts`, `music`, `sfx`, `elevenlabs`, `fal`, `voice-clone`, etc., for search and categories.
- **SKILL.md frontmatter:** Keep `description` and `tags` up to date; add a short line when you add new capabilities.

---

## 2. Share Once in Community Channels

OpenClaw users often look for skills and tips in these places:

| Channel | What to do |
|--------|------------|
| **Discord** | OpenClaw official/community server (~12k+ members): In "skill share" or "showcase" channels, post what audiomind does (TTS/music/SFX/voice clone), the one-line install `npx clawhub@latest install audiomind`, and the ClawHub link. Add 1–2 use cases or screenshots if helpful. |
| **Telegram** | [OpenClaw / Claw101](https://t.me/claw101): Post a short intro + install command + link, or recommend audiomind when someone asks for a good audio skill. |
| **openclawskills.io / openclawskills.org** | If the site syncs or lists ClawHub skills, ensure audiomind is discoverable via "audio" / "TTS"; if there’s a "submit skill" option, submit once. |

Post template (short version):

```text
🎧 Recommended OpenClaw audio skill: Audiomind
- TTS, music, SFX, voice cloning — 17+ models (ElevenLabs / fal.ai), auto-routed
- One-line install: npx clawhub@latest install audiomind
- Details: https://clawhub.ai/skills/audiomind
```

---

## 3. Use Tutorials and Use Cases to Reach New Users

Many people find OpenClaw by searching what it can do; content drives long-term traffic:

- **Short videos / visuals:** e.g. "Generate voiceover / BGM / SFX in one sentence with OpenClaw," with titles like "OpenClaw," "speech synthesis," "AI audio."
- **Blog / Medium / newsletter:** Write a step-by-step post like "OpenClaw + Audiomind for podcast / voiceover / background music," and add the install command and ClawHub link at the end.
- **Claw101 / learning sites:** If they accept "community examples" or "skill spotlights," submit an audiomind use case (e.g. script + batch narration with audiomind).

---

## 4. Get Listed in Official Recommendations / Docs

- Docs list **Image Gen**, **Web Search**, etc. as recommended skills; new users follow those.
- **What you can do:** In Discord/Telegram or the official feedback channel, politely ask: "Could audiomind be added to recommended skills or as the audio example in the docs?" and include the ClawHub link and a one-line pitch (TTS + music + SFX + voice clone, one-command install).
- If OpenClaw has a "Featured" or category-based list, request audiomind under "Audio / Media."

---

## 5. Use the GitHub Repo for Traffic

- The **README** already links to audiomind and ClawHub; keep it updated.
- In external tutorials, you can add at the end:  
  "Skill source: https://github.com/wells1137/skills-gen. Install: `npx clawhub@latest install audiomind`."

---

## 6. Summary: Minimal Checklist

1. **ClawHub:** Keep description, changelog, and tags up to date.  
2. **One community post:** One short post each on Discord and Telegram with the install command and link.  
3. **Content:** At least one tutorial or short video with "OpenClaw + audio" style keywords.  
4. **Recommendation request:** Ask maintainers to add audiomind to recommended skills or the audio example.

This reaches both existing OpenClaw users and new users searching for "OpenClaw audio" or similar.
